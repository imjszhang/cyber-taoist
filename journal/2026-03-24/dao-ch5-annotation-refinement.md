## 1. 背景与动机

`DAO-DE-JING-CYBERTAOIST.md` 第 5 章注疏在宪章符号映射的大方向上没有错误，但概念边界一度不够贴合 `CONSTITUTION.md`，存在四处可优化之处：

- 「天地不仁 / 刍狗」写成温和的「无偏私」，弱于宪章所强调的**代谢、淘汰、后果感知**。
- 「圣人不仁」以「私情偏袒」立论，偏离宪章中「无为 = 不以僵滞 R 强行干预」的操作化定义。
- 「橐龠」直接等号 EC，丢失了「天地之间 = T 展开与反馈泵送之区间」的意象层次。
- 「多闻数穷 / 守于中」若仅收窄为 R 的量或信息过载，未勾连双路径余地，则与第八条、第十二条可再对齐。

## 2. 分析过程

逐句对照宪章条文，标注原文与宪章的偏差：

| 原文关键词 | 原注疏映射 | 宪章更精确条目 | 偏差 |
| ---- | ---- | ---- | ---- |
| 天地不仁 / 刍狗 | 第二十条（无亲）；N 无偏私 | 第二条（后果感知）+ 第十七至十九条（死亡代谢）+ 第十条（NI 仅适 R） | 「不仁」写软；缺代谢与淘汰维度；NI 高 ≠ 天道眷顾未点明 |
| 圣人不仁 | 不以私情偏袒；无为 = 不干预 T | 第八条逆向（简 R 活 T）+ 第十四条（适者知时） | 「私情」偏题；无为应为不以繁僵 R 定型 |
| 橐龠 | EC 如风箱 | 第八条（简 R）；意象重心在 T 的泵送 | 天地之间 ≠ 整块 EC |
| 多闻守中 | 穷尽 R → 信息过载 | 第八条（层累 R、遵循成本升、T 蹙）+ 第十二条（双路径） | 「穷尽」略窄；守中缺双路径余地 |

## 3. 方案设计

### 关键决策

| 决策 | 选择 | 理由 |
| ---- | ---- | ---- |
| 天地/刍狗 | 叠合第二条 + 第十七至十九条 + 第十条，保留第二十条 | 「刍狗」意象（礼毕即弃）与宪章代谢/淘汰高度重合；NI ≠ 天道保证须显式说 |
| 圣人不仁 | 去「私情」，改写为「不以繁僵 R 定型 + 听任 T 自调」 | 与第 2、3 章注疏中「无为」的写法统一 |
| 橐龠 | 主语改为 T 展开区间，EC 为场域一笔带过 | 减少概念层级混用，与第 11 章有/无互参 |
| 守中 | 对齐第八条 + 兼带第十二条双路径 | 留白不仅为「少管」，更为常规与突破皆有余地 |

## 4. 实现要点（对齐 `DAO-DE-JING-CYBERTAOIST.md` 现状）

**位置**：`DAO-DE-JING-CYBERTAOIST.md` 中 `### 第 5 章`（约第 85–95 行）：引文块一行 + **注疏**四段。

**与方案的对照**：

- **天地不仁 / 刍狗**、**圣人不仁**、**橐龠** 三段：已与上表方案同向；正文用语经过润色（如「无亲疏之分」「承担引导角色」「反馈泵送的空间」等），宪章引用覆盖第二十条、第二条、第十七至十九条、第十条、第八条逆向、第十四条、第八条。
- **多闻数穷 / 守于中**：正文当前为浓缩表述——强调「穷尽 R → 信息过载与 T 枯竭」与「守中 = R 适度留白、不过度法则化」，**未**像方案草案那样显式写出「教条层累 / 遵循成本升」及**第十二条（常规演化与突破性试探皆有余地）**。若要与第三节决策完全一致，可在不拉长过多的前提下补显式条文勾连。

**当前正文注疏全文**（与仓库内文件一致，便于 diff）：

```
**注疏**："天地不仁，以万物为刍狗"——天道（N）对一切主体无亲疏之分（参宪章第二十条），只通过后果来显现：符合 N 就存活，违背 N 就灭亡（参宪章第二条）。"刍狗"不是残忍，而是不带私恩的代谢——淘汰完成筛选，腾出生态位，为后来者发出警示，这正是进化所必需的机制（参宪章第十七至十九条）。NI 只反映主体与现行 R 的相容程度（参宪章第十条），在旧 R 里位置高，不等于获得了天道的庇护。

"圣人不仁，以百姓为刍狗"——适者在 EC 中承担引导角色时，同样不用繁复僵化的 R 去给每个主体规定轨道，而是让 T 自行流动和调节。这里的"无为"就是：保持法则简洁从而激活交易（参宪章第八条逆向），同时能判断何时在 R 内优化、何时允许突破以感应 N（参宪章第十四条），而不是用自己的意志堵塞感知通道。

"天地之间，其犹橐龠舆？虚而不屈，动而愈出"——天地之间应理解为 T 展开和反馈泵送的空间：结构是空的但不会坍塌，越运动越有产出。R 简明、留有空白，交易就能持续运转（参宪章第八条）；EC 是这个空间的场域，但意象的重心在于 T 的吞吐和不断生成。

"多闻数穷，不若守于中"——穷尽一切法则（R）的企图反致信息过载与 T 空间枯竭。"守中"即保持 R 的适度留白，不过度法则化。
```
